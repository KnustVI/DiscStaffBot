TITAN'S PASS - SISTEMA DE PREMIUM
==================================================
Explicacao dos planos pagos do bot: Player Premium (por jogador) e Server
Premium (por servidor). Pagamento e feito por fora (Pix, etc.) e o tier e
concedido manualmente pelo desenvolvedor do bot - nao ha cobranca automatica
ainda.

2) SERVER PREMIUM (por servidor Discord)
------------------------------------------------

FREE

- Sistema BASICO de reporte: limite de 1 chat de reporte E 1 revisao de
  punicao (contadores SEPARADOS - pode ter os dois abertos ao mesmo tempo,
  so nao pode abrir um 2o do MESMO tipo), tempo de espera de 1h entre
  aberturas (conta pra qualquer um dos dois tipos).
- Sistema BASICO de punicao: registra a punicao normalmente, mas NAO aplica
  acoes automaticas no Discord (timeout/kick/ban - o campo existe no
  comando /strike, mas e ignorado neste tier) nem no jogo (RCON). Sem
  sistema de reputacao (pontos nao sao calculados/salvos).
- Sistema BASICO de eventos: /evento cria so a postagem no forum (com
  anexo de imagem) - sem evento agendado do Discord, sem marcar cargo de
  notificacao.
- Toda punicao com duracao MAIOR que 72h ou permanente exige aprovacao do
  cargo Supervisor (unica regra de confirmacao neste tier, ja que nao ha
  niveis de severidade relevantes).
- /config-punishments BLOQUEADO neste tier (nada a configurar, ja que
  reputacao esta desativada).

RASTREADOR - R$25/mes ("Tudo do Free +")
- Logs de jogo (alem dos logs de sistema do Free).
- Missoes mensais do Titan's Pass ativadas. (ainda nao implementado -
  "vindo em breve")
- Sistema MEDIO de reporte: limite de 3 chats de reporte E 3 revisoes de
  punicao (contadores separados), sem tempo de espera. Pontuacao de
  reputacao ativada.

3) COMO CONCEDER/REVOGAR/CONSULTAR UM TIER (uso interno, dev do bot)
------------------------------------------------
Comando: /premium-admin (restrito ao desenvolvedor do bot - nenhum admin de
servidor pode conceder premium, nem pra si mesmo, nem pros proprios
jogadores)

/premium-admin player grant usuario:<user> tier:<compy|raptor> dias:<opcional> observacao:<opcional>
/premium-admin player revoke usuario:<user>
/premium-admin player check usuario:<user>

/premium-admin guild grant servidor_id:<id> tier:<pegada|fossil> dias:<opcional> observacao:<opcional>
/premium-admin guild revoke servidor_id:<id>
/premium-admin guild check servidor_id:<id, opcional - padrao: servidor atual>

- "dias" vazio = concessao vitalicia (sem expiracao).
- servidor_id e um ID numerico digitado a mao (nao precisa estar no
  servidor no momento de conceder).
- Ao rodar "guild grant" com tier pegada/fossil, o DONO do servidor Discord
  (guild.ownerId) recebe automaticamente o Player Premium correspondente
  (Rastreador->Compy, Caçador->Raptor), pela mesma quantidade de dias, SEM
  rebaixar um tier melhor que ele ja tenha por conta propria. So funciona
  se o bot conseguir identificar o dono (precisa estar no servidor ou o
  servidor precisa ser resolvivel via client.guilds.fetch) - caso
  contrario, o comando avisa que o bonus nao foi aplicado.

Como qualquer um confere o proprio tier (sem ser o desenvolvedor):
- /perfil - mostra o Player Premium do jogador, se houver.
- /premium - mostra o catalogo completo (todos os tiers, valores, como
  adquirir), o Server Premium do servidor atual e o Player Premium de quem
  rodou o comando.


4) ATENCAO - SERVIDOR SEM TIER DEFINIDO CAI NO FREE
------------------------------------------------
Por decisao do dono do bot: qualquer servidor sem uma linha em guild_premium
(ou com o tier expirado) e tratado como FREE automaticamente - incluindo
servidores que ja usavam o bot antes desse sistema existir. Depois de
colocar esse sistema no ar, rode /premium-admin guild grant pros servidores
que devem continuar com os limites antigos (ilimitados), assim que possivel
- senao eles ficam presos nos limites do Free sem querer.


5) NENHUM COMANDO DE TERMINAL E NECESSARIO PRA USAR ISSO
------------------------------------------------
Toda concessao/revogacao de premium acontece pelo comando /premium-admin,
dentro do Discord - nao existe (nem e necessario) nenhum script de terminal
pra operacao normal deste sistema. Ver FERRAMENTAS.txt se algum dia for
criado um script auxiliar (ex: concessao em massa/emergencial).


6) AQUISICAO (PAGAMENTO)
------------------------------------------------
Hoje e 100% manual: o jogador/admin fala com o desenvolvedor do bot (via
/reportarbug, opcao Sugestao, ou contato direto), combina o pagamento (Pix,
por enquanto) e o dev concede o tier via /premium-admin.

Assim que terminarmos de separar todo o uso/comandos por tier (este
trabalho), o proximo passo e automatizar a aquisicao via Ko-fi (taxa menor
e ja disponivel pro Brasil, diferente do Discord Monetization, que ainda
nao paga desenvolvedores brasileiros).


7) BANNERS/FOOTERS DE IMAGEM E IDENTIFICACAO PADRAO DE USUARIO
------------------------------------------------
- /premium (visao geral e Server Premium) mostra um banner de topo que
  representa o tier ATUAL do servidor (assets banner_premium_server_free/
  rastreador/cacador). Todo container do /premium e do /perfil usa uma
  imagem de rodape por tier de Player Premium (footer_free/compy/raptor) NO
  LUGAR do rodape de texto padrao do bot ("Produzido por...").
- Todo container que mostra um usuario (strike, unstrike, repset, historico,
  reportchat, perfil/registrar) usa o mesmo formato padrao:
  mencao do Discord + ID Alderon (se registrado) na primeira linha, nome de
  usuario + nome do personagem na segunda, e "Conta linkada!" na terceira
  (as duas ultimas so aparecem se a pessoa tiver /registrar feito).
- Esse formato depende dos emojis de aplicacao "PotLogo" e "DiscLogo" (ver
  Discord Developer Portal > Application > Emojis) - se ainda nao tiverem
  sido sincronizados no ambiente atual, rode "npm run sync-emojis" (funciona
  so onde o .env com TOKEN/CLIENT_ID existir, tipicamente o VPS de producao)
  para os IDs reais aparecerem; ate la, o bot usa emojis unicode de reserva
  (🦖/💬) automaticamente, sem quebrar nada.
- Os titulos de cada tier no /premium usam emojis de aplicacao proprios:
  HerbSkull (tier Free/base), TapejaraSkull (tier medio - Compy/Rastreador)
  e CarniSkull (tier top - Raptor/Cacador). TapejaraSkull tambem depende de
  "npm run sync-emojis" - sem ele cai no unicode de reserva (🟡).


8) CORRECAO: /botstatus NUNCA mostrava estatisticas (bug antigo)
------------------------------------------------
O painel de "Analise de Sistemas" do /botstatus (Banco de Dados/AutoMod)
dependia de uma flag "analyticsEnabled" que nunca tinha sido definida em
PremiumSystem.GUILD_LIMITS - ou seja, NENHUM servidor via essa secao,
nem os Cacador. Corrigido nesta revisao (analyticsEnabled: true so no
Cacador) - servidores Cacador agora realmente veem essas estatisticas.
